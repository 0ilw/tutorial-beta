О проверке на StormTime. Доказательная база

На StormTime действует очень важный принцип проверок, отличающий нас от большинства других серверов – бан может выдаваться лишь за использование чит-клиента (и где возможно или требуется это выяснить – на StormTime) и в рамках 2 недель со дня проведения проверки.

В качестве исключения из правил рассматриваются лишь autobuy (topka, xorek, cycle и т. д.), automyst (на данный момент представлен лишь xorek), за которые выдаётся бан 14 дней, если они имеются на ПК игрока (если же используются – бан навсегда) и отдельные особые инжекты, рассматриваемые на Alpha или Method.

Вследствие этого на проверке помимо обнаружения читов требуется ещё и выяснить, в какое время и где они использовались (чтобы определить сервер, нужно уметь работать с логами, см. подробнее: [нажми сюда]).

Это требование распространяется и на чит-клиенты, расположенные в папках игры («внутренние»), и на читы, что запускаются извне этих папок («внешние»).

При проверке также учитывается и то, были ли у игрока, которого вы проверяете, баны до вас. Для этого необходимо использовать две важные команды — /history (посмотреть баны на вызванном аккаунте) и /dupeip (баны на других аккаунтах).

⚠️ Также бан выдаётся за очистку той или иной информации, удаление или перемещение чит-клиента или запрещённых модификаций. Чтобы выдать наказание, необходимо это доказать. Используется программа JournalTrace и сравнение раздела mods в игре и папки mods.
Страница 3

/history & /dupeip

/dupeip – используется для того, чтобы посмотреть, есть ли у игрока другие аккаунты (твинки).
/history – используется для того, чтобы посмотреть, были ли баны на определённом аккаунте, будь им вызванный на проверку, будь им названный игроком.

❗ Не важно, за какой подпункт (4.3, 4.3.1 или 4.3.2) был выдан бан. Он учитывается в любом случае.
📌 Пример: Inertia + BleachHack

Вызов игрока Capi 11 мая 2023
/history capi → банов нет
Игрок сказал, что был на аккаунте Tink
Обнаружено: играл с Inertia 5 мая на StormTime
/history tink → бан 6 мая (п.4.3.1)
Также играл с BleachHack 9 мая → Бан

📌 Пример: Impact + Aristois

Вызов игрока Capi 11 мая 2023
/dupeip capi → нет твинков
Обнаружено: играл с Impact 5 мая на StormTime
/history capi → бан 6 мая (п.4.3.2)

Страница 4

Logs (логи). Как с ними работать

Логи используются, чтобы точно подтвердить использование чита или запрещённого мода и узнать, на каком сервере это произошло (нас интересует StormTime).

  • Для открытия архива с логом (.gz) нужен архиватор.
  • Нужно предварительно знать дату и время использования чита/мода.
  • Рекомендуется перекидывать лог себе (Discord/AnyDesk).

В логе ищем через Ctrl+F:

  • А. StormTime (приветствие «Приветствуем на StormTime»)
  • Б. Название чита/мода
  • В. Setting user: (ник, с которого использовалось)
🔍 Убирайте галочку «С учётом регистра». Если вызванный аккаунт и ник в логе различаются, и у игрока не было банов ни там, ни на твинке — бан выдаётся на оба аккаунта.

Классификация чит-клиентов. Запрещённые модификации

Внутренние чит-клиенты

Располагаются в папках игры (.minecraft, .tlauncher), в versions и mods.

  • Чит-версия (jar-version): Impact, Aristois, Celestial Free, Rich Free, Matix, Destroy Squad и др.
  • Чит-модификация (jar-mod): Aristois, Impact (Forge), Inertia, Meteor, Bleach, MatHax, Wurst.
  • Ghost-клиенты: PseudoShulker, Sweet/TutorialClient, ClowdyLite, Bozar, NekoClient.
  • Hitboxes (HB): PseudoNeat, Hb in Neat, Hb in BetterHUD, ShulkerBoxTooltip и др.

Внешние чит-клиенты

  • .exe-inject: Cortex Legit, DreamPool
  • .dll-inject: Vertzah, TopkaAutoBuy v2 (инжект через Process Hacker 2)
  • .exe-launcher: Akrien Beta, CelestialRecode, Expensive, Nursultan, DeadCode, Wild, Excellent

Запрещённые модификации

Полный список: https://forum.funtime.su/modifications

Страница 8

Проверка игры. Папка mods. Игровые файлы

Первое — проверка игры: включены ли функции (если да — сразу бан). Далее F3 (название версии, может быть Impact/Aristois). F3+b — размер хитбоксов. X-Ray: Esc → Настройки → Наборы ресурсов. Если есть X-Ray — /revise finish XRay (п.4.3.2).

Страница 9

Проверка лаунчера (диспетчер задач)

Проверьте диспетчер задач: если версия 1.16.5 OptiFine/vanilla, а процесс называется не javaw.exe, а например expensive.exe или случайный набор — это .exe-launcher чит.

Страница 10

LabyMod / Lunar Client

LabyMod 4: проверять дату изменения .json в configs/customhurtcam — если совпадает с заморозкой → бан (/revise finish Прочее(14дней)). LabyMod 3: активный NoHurtCam → предупреждение/блокировка. Lunar Client: если HurtCam не заблокирован — предупреждение.

Страница 11–12

Перемещение мода из папки mods

Сравнить раздел Mods в игре и папку mods. Если мод отсутствует в папке, но есть в разделе — бан за попытку обхода: /revise finish Удаление(варн). Если раздел недоступен — JournalTrace.

В папке mods могут быть чит-моды (Forge/Fabric), ghost-клиенты, вписанные хитбоксы. Используйте таблицу весов (доступ через сотрудников).

Страница 14–15

Проверка папок с игровыми файлами

  • baritone: ищем mc.stormtime.su или play.stormtime.su, внутри файлы r1.0.bcr — дата изменения = время последнего использования чита.
  • Папка с настройками читов: дата изменения .cfg/.json = дата использования.
  • versions: дата .json = дата последнего запуска чит-версии.
  • config: дата изменения файла = дата использования мода.
  • Перед проверкой включить показ скрытых файлов (control folders). Проверить дату изменения resourcepacks и versions — если совпадает с заморозкой → бан за очистку.
Страница 16

Корзина ($Recycle.Bin)

Дата изменения корзины совпадает с заморозкой → /revise finish Очистка-корзины. Проверять через C:\$Recycle.Bin или %systemdrive%\$Recycle.Bin.

Страница 17

Проверка C:\, Downloads, Desktop

В C:\ искать папки exe-launcher’ов (AkrienAntiLeak, Celka, Expensive и т.д.), внутри смотреть baritone и logs. В Downloads/Desktop — лоадеры Cortex (дата Launch versions), .dll, .bat (зовём Alpha), .jar >1600 КБ (Doomsday).

Страница 18–20

Everything. ExecutedProgramsList

Everything — поиск файлов по размерам. Фильтры: .exe size:huge , .jar size:1.8mb-2.4mb. Найденный Cortex — проверить время; Doomsday — банить; .dll — звать Alpha.

Перед работой подключить все NTFS-диски в настройках Everything.

ExecutedProgramsList

  • Сортировка по «Last Executed On» (запуски в день проверки). Обращать внимание на пустые поля описания.
  • Двойная сортировка: по времени и Product/Company name.
  • Сортировка по размеру (от большего) — для поиска Cortex.
Страница 24–27

JournalTrace — работа с действиями над файлами

Отображает перемещение, переименование, удаление. Два режима: Data Grid (для mods) и Directory Tree (для C:\, корзины).

  • Перемещение мода из mods → Data Grid → путь к mods → file delete / rename.
  • Очистка C:\ → Directory Tree → C:\ file delete.
  • Удаление DoomsDay (.jar) → Data Grid → .jar file delete.
  • Удаление через корзину → Directory Tree → $Recycle.Bin → S-1-5... → Rename.
📌 Алгоритм: выбрать диск → Scan → Layout (Data Grid или Directory Tree). ПКМ по файлу → Show file information. Если после запуска игры удалён .exe — консультироваться с Alpha.
